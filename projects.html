Handlingo — ASL Learning App (React, Next.js, TypeScript)

Role: Co-founder · Full-stack / Computer Vision Engineer
Tech: React, Next.js, TypeScript, Supabase, TensorFlow.js, MediaPipe-style landmark detection

Problem

ASL beginners often struggle to practice consistently because they don’t get clear, immediate feedback on their signing. Most tools rely on server-side models, which causes noticeable latency and makes real-time sign recognition feel laggy or inaccurate.

Solution

Designed and built an ASL learning app that performs real-time hand-landmark detection directly in the browser. The model runs fully client-side using TensorFlow.js to keep inference fast, stable, and cost-free. Integrated Supabase to handle authentication, user progress tracking, and profile data, allowing learners to see improvement over time.

Challenges

Optimizing the hand-landmark model so it ran smoothly on all laptops without dropping frames was a key challenge. Balancing prediction accuracy with responsiveness required multiple iterations. Another challenge was structuring Supabase data so progress updates stayed consistent across devices.

Outcomes & Metrics

  • Achieved sub-100ms prediction latency in the browser.
  • Eliminated dependency on external ML endpoints, removing model-API latency.
  • Created a beginner-friendly UX that encourages repeated practice and consistent progress tracking.

Process / Learnings

Learned how to manage real-time prediction loops, tune front-end performance for computer vision workloads, and design a lightweight backend that stays in sync with client-side ML. The project strengthened skills in model evaluation, UI/UX for real-time apps, and building end-to-end learning experiences.

AI NPCs — Contextual game dialogue (Python, OpenAI)

Role: Prototype lead · Tech: Python, GPT

Problem

Game NPCs often feel repetitive; we wanted NPCs that can respond dynamically to player actions without heavy manual scripting.

Solution

Built a dialogue pipeline that conditions prompts on recent player actions and NPC state, using GPT to generate context-aware replies. Implemented throttling and caching to keep costs low.

Challenges

Balancing prompt length for context vs API cost and ensuring replies stayed in-character.

Outcomes & Metrics

  • Prototype tested with hackathon users — participants rated NPC realism 4.2/5.
  • Implemented caching that reduced API calls by ~40% during sessions.

Process / Learnings

Focused on prompt engineering and building a small state-management layer to keep NPC responses coherent. Learned practical cost control for LLM-backed features.

NASA Space Explorer — API-Driven Gallery (HTML, CSS, JavaScript)

Role: Front-end Developer · Tech: JavaScript (vanilla), NASA APOD API, HTML/CSS, async data pipelines

Problem

Users exploring NASA’s APOD feed often struggle with inconsistent media, missing fields, and slow network conditions. Many demos break when entries are videos, or when metadata is incomplete.

Solution

Built an interactive space gallery that gracefully handles mixed media types, fallback fields, and network errors. Added a responsive modal system, keyboard-accessible navigation, video embedding, and a “Did you know?” fact generator for engagement.

Challenges

Handling APOD entries with inconsistent structures, supporting YouTube/Vimeo videos inside a modal, and managing large HD image loads without blocking the UI.

Outcomes & Metrics

  • 100% of APOD entries load without breaking the UI.
  • Lazy-loading reduced initial load time significantly on mobile.
  • Interactive hover and modal system increased user engagement in tests.

Process / Learnings

Gained experience with defensive coding for real APIs, building fully responsive components without frameworks, and designing accessible UI patterns.

Intel Event Check-In — QR Attendance System (React, Node.js)

Role: Front-end Developer · Tech: React, Node.js, QR generation, localStorage, responsive UI

Problem

Events needed a fast, simple check-in tool where participants could scan a code, verify attendance, and avoid manual spreadsheets or slow admin flows.

Solution

Built a responsive web application that generates unique QR codes and handles check-in flows. Optimized UI to make scanning fast on mobile devices and used local storage to maintain state between sessions.

Challenges

Ensuring QR codes scaled well across devices, managing inconsistent camera permissions, and building an intuitive flow for non-technical attendees.

Outcomes & Metrics

  • Check-in process reduced to under 3 seconds per user.
  • Fully functional mobile-first interface for real-world use.
  • Strong reliability even with unstable Wi-Fi.

Process / Learnings

Gained experience with QR libraries, user-flow design, and building tools for real-world constraints.
